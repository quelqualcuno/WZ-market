# ZeroMarket

**ZeroMarket** is a full-stack virtual item economy platform inspired by the Roblox game World // Zero. It provides a centralised shop, peer-to-peer marketplace, auction system, reputation management and advanced economic analytics for virtual items.

## Features

### ✅ MVP (v1.0)
- User registration & JWT authentication
- Centralised shop with 20+ seeded items
- Purchase items (balance deducted, inventory updated)
- Price history tracking per item

### 🔄 v2.0
- **Marketplace P2P** – list items for direct sale with escrow protection
- **Persistent offers** – offers stay active until accepted or cancelled
- Seller/buyer reputation with star ratings

### 🔥 v3.0
- **Auction system** – create timed auctions, place bids, automatic extension, buyout
- **Economic analytics** – price charts (7-day / 30-day), volatility, volume
- **Anti-inflation engine** – automatic price adjustment based on global stock levels
- **Admin panel** – user management, item restocking, transaction logs

### ✨ Quality of Life
- Dark / Light mode toggle
- Responsive design
- Item rarity badges (common → legendary)
- Portfolio P&L tracking in inventory
- Live auction countdown timers

## Getting Started

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) 24+
- [Docker Compose](https://docs.docker.com/compose/) v2+

### Quick Start (Docker)

```bash
# 1. Clone the repository
git clone https://github.com/quelqualcuno/WZ-market.git
cd WZ-market

# 2. Copy and configure environment variables
cp .env.example .env
# Edit .env and set a strong SECRET_KEY

# 3. Start all services
docker compose up --build

# 4. Open in browser
# Frontend: http://localhost
# Backend API docs: http://localhost:8000/docs
```

On first startup the backend automatically:
1. Creates all database tables
2. Seeds 5 categories and 22 items
3. Creates an admin user (`admin` / `admin123`)

### Local Development

#### Backend

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Set environment variables (or create .env in backend/)
export DATABASE_URL="postgresql+asyncpg://user:pass@localhost:5432/zeromarket"
export SECRET_KEY="your-secret-key-at-least-32-chars"

# Run migrations
alembic upgrade head

# Start server
uvicorn app.main:app --reload
# API available at http://localhost:8000
# Swagger UI at http://localhost:8000/docs
```

#### Frontend

```bash
cd frontend

# Install dependencies
npm install

# Start dev server (proxies /api to http://localhost:8000)
# Update vite.config.js target if backend is not on localhost:8000
npm run dev
# App available at http://localhost:5173
```

## API Documentation

The full interactive API documentation is auto-generated by FastAPI:

- **Swagger UI**: `http://localhost:8000/docs`
- **ReDoc**: `http://localhost:8000/redoc`

### Main Endpoints

| Method | Path | Description |
|---|---|---|
| POST | `/api/users/register` | Register new user |
| POST | `/api/login` | Login, returns JWT |
| GET | `/api/users/me` | Current user profile |
| GET | `/api/items` | List items (filterable) |
| GET | `/api/items/{id}` | Item detail + price history |
| POST | `/api/orders` | Buy item from shop |
| GET | `/api/inventory` | Current user inventory |
| GET | `/api/offers` | List marketplace offers |
| POST | `/api/offers` | Create P2P listing |
| POST | `/api/offers/{id}/buy` | Purchase offer (escrow) |
| GET | `/api/auctions` | List auctions |
| POST | `/api/auctions` | Create auction |
| POST | `/api/auctions/{id}/bid` | Place bid |
| POST | `/api/ratings` | Submit user rating |
| GET | `/api/admin/dashboard` | Admin stats |

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `DATABASE_URL` | *(required)* | Async PostgreSQL connection string |
| `SECRET_KEY` | `changeme-...` | JWT signing secret (min 32 chars) |
| `ALGORITHM` | `HS256` | JWT algorithm |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `1440` | Token expiry (24 hours) |
| `POSTGRES_USER` | `zeromarket` | PostgreSQL user (Docker) |
| `POSTGRES_PASSWORD` | `zeromarket_pass` | PostgreSQL password (Docker) |
| `POSTGRES_DB` | `zeromarket` | PostgreSQL database (Docker) |
